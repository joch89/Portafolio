Proyectos de Machine Learning

API de Predicción Breast Cancer

Descripción: Modelo de Machine Learning entrenado con el dataset Breast Cancer (Wisconsin) y una API REST en Flask para realizar predicciones. El proyecto está dockerizado y cuenta con un workflow CI/CD en GitHub Actions.

Objetivos:

  • Proveer una API REST para predecir benigno o maligno a partir de 30 características de tejido mamario.
  • Contener el modelo y la API en Docker para ejecución sencilla en cualquier máquina.
  • Automatizar pruebas y despliegue mediante CI/CD en GitHub Actions.

Tecnologías: Python, Flask, Docker, Scikit-Learn, GitHub Actions

Resultados:

  • API funcional con endpoints GET / y POST /predict.
  • Imagen Docker con la API lista para usar.
  • Workflow CI/CD que construye, prueba y publica la imagen automáticamente.

Reflexión: Aprendí a integrar un modelo de ML con una API REST, dockerizar la aplicación y automatizar el despliegue con CI/CD, asegurando reproducibilidad y escalabilidad.

Análisis de Patada de Costado

Descripción: Proyecto de visión por computadora con MediaPipe y OpenCV para detectar, registrar y analizar el movimiento de una patada de costado (yoko geri) en Taekwondo.

Objetivos:

  • Evaluar la postura y ángulos articulares (cadera, rodilla).
  • Detectar automáticamente inicio y fin de patadas.
  • Generar métricas de velocidad, aceleración y jerk angular.
  • Proporcionar feedback visual inmediato mediante video con métricas superpuestas.
  • Crear un archivo CSV con todos los landmarks y métricas para análisis numérico flexible y extensible.

Tecnologías: Python, MediaPipe (CNN), OpenCV, Numpy, Pandas, Matplotlib, SciPy

Resultados:

  • Video procesado con landmarks y contador de patadas.
  • Gráficos de ángulo, velocidad, aceleración y jerk.
  • CSV con métricas de cada patada para análisis comparativo.

Reflexión: Aprendí sobre detección de landmarks, análisis cinemático y procesamiento de video en tiempo real, integrando datos cuantitativos y visuales para mejorar la técnica deportiva.

Segmentación y Detección de Anomalías en Pacientes Crónicos

Descripción: Proyecto de **Aprendizaje No Supervisado** para segmentar pacientes del **Pima Indians Diabetes Dataset** e identificar perfiles atípicos (anomalías). El enfoque se centró en evaluar la interpretabilidad clínica de los *clusters* y la robustez de los *outliers* detectados.

Objetivos:

  • Aplicar y comparar distintos algoritmos de Clustering (K-Means, DBSCAN, HDBSCAN) en un dataset de salud.
  • Utilizar técnicas de Reducción de Dimensionalidad (PCA, UMAP, t-SNE).
  • Implementar modelos de Detección de Anomalías (Isolation Forest, One-Class SVM).
  • Evaluar los resultados del clustering.
  • Justificar la elección final del modelo de clustering en base a las métricas y la interpretabilidad clínica de los segmentos.

Tecnologías: Python, Scikit-learn, UMAP, HDBSCAN, Pandas, Numpy, Matplotlib, Seaborn

Resultados:

  • Identificación de grupos de pacientes con características clínicas similares (ej., alto IMC + alta glucosa).
  • Visualización en 2D del clustering de pacientes, demostrando que UMAP + DBSCAN/HDBSCAN supera a PCA.
  • Un conjunto claro de pacientes anómalos cuyo perfil clínico fue validado por múltiples modelos (Isolation Forest y ruido de DBSCAN).
  • Análisis comparativo justificando la selección de hiperparámetros del modelo final.

Reflexión: El proyecto profundizó mi entendimiento en la aplicación de modelos de *clustering* y detección de anomalías en escenarios clínicos. Aprendí a priorizar la **interpretabilidad** sobre la simple optimización métrica en un contexto de toma de decisiones en salud.
